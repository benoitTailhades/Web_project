Optimization of Regression Models in Distributed Computing

Research Department, Big Data Laboratory

Context of Massively Parallel Architectures

Training machine learning models on massive datasets requires decentralized computing architectures. When a feature matrix $X$ of dimension $n \times p$ exceeds the RAM capacity of a single node, data splitting is required. The approach studied here relies on Data Parallelism to distribute gradient computation across multiple processing units.

Derivation and Stochastic Gradient Descent

In a standard linear regression problem, the objective is to minimize the mean squared error cost function. Let $w$ be the weight vector. The function is formally written as $J(w) = \frac{1}{2m} \sum_{i=1}^{m} (h_w(x^{(i)}) - y^{(i)})^2$. In a distributed environment, the exact computation of the partial derivative for each parameter is bandwidth-intensive. Therefore, we apply a stochastic approximation method where each node computes a local gradient on a mini-batch before synchronizing with the master server.

Experimental Results and Perspectives

Tests conducted on a 16-node cluster demonstrate that the learning rate must be dynamically adjusted based on the variance of local gradients. By using a covariance matrix to evaluate the dispersion of updates, we stabilized the algorithm. This methodology provides a robust foundation for future deployments of deep neural networks in Edge Computing.
